Section 2: Project Directory Structure & Data Models

How agent-team-hub organizes files in each project

Project Directory Structure
my-project/
├── .hub/                    # Hub config (per-project)
│   ├── config.json          # Project settings
│   └── agents/              # Agent definitions
│       ├── planner.md
│       ├── backend.md
│       └── frontend.md
│
├── docs/                    # Markdown documents
│   ├── plans/
│   │   └── context.md
│   ├── develop/
│   │   ├── architecture.md
│   │   └── backend-spec.md
│   └── research/
│       └── tech-research.md
│
├── issues/                  # Issue tracking
│   ├── _index.json          # {nextId, metadata}
│   ├── ISS-001.json
│   ├── ISS-002.json
│   └── archive/
│       └── ISS-000.json
│
├── data/                    # Planning data (JSON)
│   ├── prd.json
│   ├── features.json
│   ├── roles.json
│   └── userflow.json
│
└── .claude/                 # Claude Code config
    └── skills/
        └── hub.md           # Hub skill for agents

  "id": "ISS-001",

  "status": "open|in-progress|
             resolved|closed",
  "priority": "critical|high|
               medium|low",
  "type": "task|bug|feature|
           question|decision",
  "assignee": "string",
  "assignees": ["string"],
  "reporter": "string",
  "labels": ["string"],
  "relatedFiles": ["path"],
  "relatedIds": ["F-001"],

    "author": "string",

Document Frontmatter Schema

---
title: "Document Title"              # Required
description: "Brief description"     # Optional
author: "agent-name or human"        # Required
emoji: "📝"                          # Optional (display icon)
type: "plan | develop | research"    # Required (maps to menu category)
references: ["docs/...", "data/..."] # Optional (related files)
createdAt: "2026-03-19"              # Required
---
